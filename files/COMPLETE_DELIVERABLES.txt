📊 WHAT THE SYSTEM DOES
───────────────────────────────────────────────────────────────────────────────

INPUT (Raw Articles):
  → Static: 16 pre-ingested articles
  → Real-time: 16–100 articles per cycle from 5 sources
     • NewsAPI (newsapi.org)
     • RSS Feeds (FoodBev, Reuters, FMCG Newsdesk)
     • SEC EDGAR filings (8-K, 425)
     • Google News
     • Press Release sites

PIPELINE (4 Stages):
  
  STAGE 1: INGESTION
  └─ Parse raw articles, extract structure
     Input: 16–100 articles | Output: 16–100 structured records

  STAGE 2: DEDUPLICATION
  └─ Remove duplicate coverage of same deal
     • Pass A: Exact group match (acquirer + target)
     • Pass B: Headline similarity ≥ 0.75
     Input: 16–100 | Output: 14–80 unique deals

  STAGE 3: SCORING
  └─ Assess relevance (FMCG focus) & credibility (source tier)
     • Relevance: 0–10 (keyword density, category match)
     • Credibility: 0–10 (T1=SEC/official, T2=trade, T3=blog)
     • Composite: 60% relevance + 40% credibility
     • Filter: Keep only ≥ 5.5 / 10
     Input: 14–80 | Output: 12–76 passing deals

  STAGE 4: NEWSLETTER DRAFT
  └─ Auto-generate structured newsletter
     • 5 sections: Mega (>$10B) / Strategic ($1–10B) / Bolt-on / PE / Failed
     • Key themes (5 auto-extracted)
     • Pipeline transparency block
     • Export: XLSX, DOCX, CSV, JSON, TXT

OUTPUT (Professional Documents):
  → Excel workbook (4 sheets: dashboard, deals, pipeline log, chart)
  → Word document (styled sections, themes, transparency)
  → CSV & JSON (for downstream integrations)
  → Plain text (for email/slack)

🤔 COMMON QUESTIONS
───────────────────────────────────────────────────────────────────────────────

Q: Do I need an API key to run this?
A: No. Static version works with no setup. Real-time works better with
   NewsAPI key (free tier available at newsapi.org).

Q: How fresh is the data?
A: Static = pre-computed. Real-time = 1–5 minutes old (depends on source).

Q: Can I schedule this to run automatically?
A: Yes. Use cron (Linux), Task Scheduler (Windows), or Lambda (AWS).

Q: What if I want to integrate with Slack/Email?
A: Read outputs (newsletter_draft.txt or XLSX/DOCX) and integrate downstream.

Q: Is this accurate for making investment decisions?
A: No. This is a research/monitoring tool, not investment advice.
   All outputs carry "for informational purposes only" disclaimers.

Q: Can I modify the scoring logic?
A: Yes. Edit FMCG_KEYWORDS, CREDIBILITY_TIERS, or the scoring functions
   in pipeline.py or pipeline_hybrid.py.
