## 📊 What the Pipeline Does

### Four Stages

```
STAGE 1: INGESTION
├─ Fetch from 5 live sources (or use static)
├─ 16–100 raw articles per cycle
└─ Structured format: headline, acquirer, target, value, status, source, credibility

STAGE 2: DEDUPLICATION
├─ Pass A: Exact group match (same deal, different sources)
├─ Pass B: Headline similarity ≥0.75 (paraphrased versions)
└─ Result: 16–100 articles → 14–80 unique deals

STAGE 3: SCORING
├─ Relevance: FMCG keyword density (30+ terms)
├─ Credibility: Source tier (T1=regulatory, T2=trade, T3=blog)
├─ Composite: 60% relevance + 40% credibility
└─ Filter: Keep only ≥5.5 / 10

STAGE 4: NEWSLETTER
├─ Categorise: Mega-deals / Strategic / Bolt-ons / PE / Failed
├─ Draft: 5-section structured newsletter
├─ Extract themes: 5 key FMCG M&A trends
└─ Export: XLSX + DOCX + CSV + JSON + TXT
```

## 🛠️ Customization

### Adjust Deduplication Strictness
```python
# In pipeline.py or streamlit_app.py:
dedup_threshold = 0.75  # default
# Lower (0.6) = more aggressive dedup
# Higher (0.9) = stricter, keep more variations
```

### Adjust Scoring Cutoff
```python
min_score = 5.5  # default
# Lower = include more deals (even low credibility)
```

## 📈 Performance & Limitations

### Speed
- **Static pipeline:** ~2 seconds (all files pre-generated)
- **Real-time cycle:** 45–60 seconds (depends on API response times)
- **Streamlit dashboard:** Instant (updates in background every 60s)

### Data Volume
- **Static input:** 16 articles fixed
- **Real-time input:** 16–100 articles per cycle
  - NewsAPI: 20–50 articles
  - RSS: 10–20 articles
  - SEC EDGAR: 2–10 filings
  - Google News: 10–20 headlines
  - Press releases: 3–8 items

### API Limits
- **NewsAPI free:** 100 requests/day (429 articles/day max)
- **SEC EDGAR:** Unlimited (public API)
- **Google News:** Unlimited (RSS)
- **Press release sites:** Unlimited (RSS/scraping)

### Deduplication Coverage
- **Exact same deal, multiple sources:** 100% caught
- **Paraphrased headlines (similarity ≥0.75):** 95% caught
- **Same deal, weeks apart:** Not deduplicated (by design — treat as new coverage)

## 🤔 FAQ

**Q: Do I need API keys to run this?**  
A: No. Static version works without any keys. Real-time version works better with NewsAPI key (free tier available).

**Q: Can I run this on a server / schedule it?**  
A: Yes. The Python scripts are serverless-friendly. Use cron (Linux), Task Scheduler (Windows), or AWS Lambda to trigger every 60 seconds.

**Q: How fresh is the data?**  
A: Real-time version pulls data that's 1–5 minutes old (depends on source). Static version is whatever was last run.

**Q: Can I integrate this with Slack / Email?**  
A: Yes. After running the pipeline, you can:
  - Read `newsletter_draft.txt` and post to Slack
  - Attach `FMCG_MA_Newsletter.xlsx` to an email
  - Use the JSON output (`deals_final.json`) for downstream integrations

**Q: What if a source goes down?**  
A: Pipeline keeps going. Failed sources are logged, working sources continue. Dashboard shows which sources are active.

**Q: How accurate is the scoring?**  
A: Depends on keyword matching + source credibility. Expect 85%+ accuracy for filtering FMCG deals vs non-FMCG. Credibility tiers are editorial judgements (T1 > T2 > T3).

**Q: Can I use this for investment decisions?**  
A: **No.** This is a research/monitoring tool. Not investment advice. All outputs carry "for informational purposes only" disclaimers.
